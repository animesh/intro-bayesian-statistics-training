Robust tt-test

Stan

To make the tt-test more robust, model the data with a tt-distribution Kruschke 2013

  • Group 1: h∼Tν(μ,σ2)h∼Tν(μ,σ2)
  • Group 2: h∼Tν(μ+β,σ2+γ)h∼Tν(μ+β,σ2+γ)

 

Or more compactly

h∼Tν(μ,σ2)h∼Tν(μ,σ2)
with

μ=β0+β1∗Groupμ=β0+β1∗Group
σ=γ0+γ1∗Groupσ=γ0+γ1∗Group
and the additional parameter νν

Just replace the gaussian family by student in brms

What have we neglected so far?

Priors

Stan

The priors! Let’s extract them with the prior_summary(…) function

  • The brms defaults are
prior_summary(brmfit_robust)[, -c(4:5, 7)] # hide some columns for display
##                   prior     class coef  dpar lb ub  source
##                  (flat)         b                  default
##                  (flat)         b SexM             default
##                  (flat)         b      sigma       default
##                  (flat)         b SexM sigma       default
##  student_t(3, 167, 8.2) Intercept                  default
##    student_t(3, 0, 2.5) Intercept      sigma       default
##           gamma(2, 0.1)        nu             1    default
  • Uniformative for the regression coefficients β1β1, γ1γ1
  • Intercepts are Student-tt with parameters estimated from the data
  • sigma has a log-link, so its priors are on log(σ)log⁡(σ)
  • Wide Gamma for νν with lower-bound of 1

What should we use?

Prior selection

It’s an open choice, but we should aim to use information we might know.

  • Heights are around 1.6m
  • typical variations are tens of cm
  • can be below 1m, above 2m

Let’s stick with Student-tt with low degrees of freedom (heavy tails)

  • something like student_t(3, 0, 20) for β1β1

Variation in sds could be up to a factor 2

  • something like student_t(3, 0, 0.5) for γ1γ1 (with the log-link)

Brms syntax:

brmfit_robust_p <- brm(bf(Height ~ Sex, sigma ~ Sex), family = student, 
                       prior = prior(student_t(3, 0, 20), class = "b", coef = "SexM") +
                       prior(student_t(3, 0, 0.5), class = "b", coef = "SexM", dpar = "sigma"),
                       swiss_army_df)

Updated prior

We wouldn’t expect sds below about 1cm

  • simple to impose a lower bound (say 0 with log-link)
  • or choose positive distribution (say gamma)

We also might not expect such a large change in sds between conditions

  • lower sd for γ1γ1 to student_t(3, 0, 0.2)
brmfit_robust_prior2 <- brm(bf(Height ~ Sex, sigma ~ Sex), family = student, 
                        prior = prior(student_t(3, 0, 20), class = "b", coef = "SexM") +
                        prior(student_t(3, 0, 0.2), class = "b", coef = "SexM", dpar = "sigma") + # lower sd
                        prior(student_t(3, 0, 2.5), class = "Intercept", dpar = "sigma", lb = 0), # bound
                        sample_prior = "only", swiss_army_df) # prior samples
brmfit_robust_p2 <- brm(bf(Height ~ Sex, sigma ~ Sex), family = student,

Summary

With a tt-test parametrised like lm

  • we can keep building up more complicated models
  • equal variance, unequal variance, robust, ……

With brms can easily run MCMC-based sampling

  • regression formula syntax bf(… ~ …)
  • specify distributional assumptions with family

But default priors

  • often flat or unrealistic
  • can check by sampling from prior

But we can (should) specify sensible priors for fully Bayesian inference

  • prior(…) function
