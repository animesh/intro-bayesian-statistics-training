Admission

Question: How do we handle predicting a binary response?

From binary to continous to likelihood

Answer: We predict the probability of the binary value being 1.

Let us say for each student SiSi with Admission value Ai∈{0,1}Ai∈{0,1}

  • we predict as ρˆiρ^i the probability of SiSi being admitted

ρˆi=P(Ai=1)ρ^i=P(Ai=1)

Question: What is the likelihood of AiAi?

Answer:

Li={1−ρˆiρˆiAi=0Ai=1Li={1−ρ^iAi=0ρ^iAi=1

Question: What is the log-likelihood of AiAi?

Answer: We take logs and use AiAi as an indicator function

li=Ailog(ρˆi)+(1−Ai)log(1−ρˆi)li=Ailog⁡(ρ^i)+(1−Ai)log⁡(1−ρ^i)

Log-likelihood

Log-likelihood of all the data is then

l=∑i[Ailog(ρˆi)+(1−Ai)log(1−ρˆi)]l=∑i[Ailog⁡(ρ^i)+(1−Ai)log⁡(1−ρ^i)]
We want to maximise this likelihood

  • when we predict ρˆρ^ from the covariates

When we regress

ρ∼GRE+GPAρ∼GRE+GPA

Question: What’s the problem here?

Answer: ρρ has to be between 0 and 1

Question: How do we ensure this?

Answer: Transform it.

Logit

A common choice is logit()

logit(ρ)=log(ρ1−ρ)logit(ρ)=log⁡(ρ1−ρ)

  • rotational symmetry around ρ=12ρ=12

 

Question: What is the ρ1−ρρ1−ρ term?

Answer: ρ1−ρρ1−ρ is the odds.

Question: For ρ=13ρ=13 what are the odds?

Answer: 1212. In betting odds, would be 2 to 1 (against)

Logistic regression

We have a binary response variable: yy

→→ model underlying probability of being 1: ρρ

→→ transform to odds: ρ1−ρρ1−ρ

→→ transform with log: log(ρ1−ρ)log⁡(ρ1−ρ)

→→ regress log odds on covariates:

log(ρ1−ρ)=β0+β1x1+…log⁡(ρ1−ρ)=β0+β1x1+…

To learn regression coefficients

→→ predict log odds with regression

→→ predicted probabilities ρˆρ^

→→ maximise (log-)likelihood ll numerically.

Question: How much would the predicted odds of being Admitted change if we increase the GRE by 0.4 and the GPA by 0.2?

Answer: by a factor of exp(0.4 * 0.2691 + 0.2 * 0.7547) = 1.2951.

Remember we regress the log odds on the covariates

log(ρ1−ρ)∼β0+β1x1+…log⁡(ρ1−ρ)∼β0+β1x1+…

  • change in the covariates
  • leads to a linear change in log odds

Exponentiate the linear change

  • multiplicative factor in the odds

Bayesian logistic regression

Stan

For the Bayesian version with brms, similar syntax with family=bernoulli

brm_logistic <- brm(Admission ~ GRE + GPA, family = bernoulli, 
                prior = prior(student_t(3, 0, 1), class = "b"), # sets for both beta
                admiss_data) # run the model

Bernoulli is binomial with 1 trial, hence syntax (each subject is 1 row)

  • for collated data need family=binomial and success | trials syntax
  • →→ see Exercises 8

Trace plots look good, as we would expect for a small model:

Posterior probability of admission

Stan

Posterior predictions with posterior_predict and posterior_linpred

posterior_predict

  • provides posterior samples in the outcome binary space

posterior_linpred

  • provides posterior samples in the (logit-transformed) regression space with transform = FALSE
  • or probability-space with transform = TRUE

For our particular “student”, let’s get samples in the probability space:

new_data <- data.frame(GRE = 5.5, GPA = 3.5)
new_data_prob <- posterior_linpred(brm_logistic, 
                  new_data, transform = TRUE)

GRE

Question: Is there a premium for getting a perfect score?

Bayesian premium estimate

Stan

Add perfect to the Bayesian model with brms

brm_logist_prem <- brm(Admission ~ GRE + GPA + perfect, family = bernoulli, # add perfect
                   prior = prior(student_t(3, 0, 1), class = "b"), # sets for all beta
                   admiss_data) # run the model
mcmc_plot(brm_logist_prem, variable = "b_per",
    regex = TRUE, prob_outer = 0.95, 
    type = "intervals") + geom_vline(
      xintercept = 0, color = "darkorange")

 

  • Posterior of perfect score premium is typically positive, but a lot of uncertainty!

Summary

For binary response variables

  • logistic regression
  • model log odds as a function of covariates

The exponential of the regression coefficients

  • related to multiplicative change in the odds

With brms can again easily run Bayesian logistic regression

  • with the same regression formula syntax bf(… ~ …)
  • but with family=bernoulli

And a slightly different syntax for family=binomial
